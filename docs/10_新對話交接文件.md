# 二、已完成重點

## 共同檢查清單

目前仍使用：

```text
Trip JSON checklistData
↓
ChecklistPage
↓
useChecklistState
↓
checklistService
↓
checklistStorage
↓
localStorage
```

目前能力：

- 勾選狀態依 `tripId` 保存。
- F5 後保留勾選狀態。
- 不同 Trip 各自保留進度。
- `guest` / `user` 可見不可勾選。
- `trip_editor` / `super_admin` 可勾選。

尚未完成：

- 共同檢查清單雲端同步。
- Trip JSON seed 初始化到 Supabase shared checklist。
- 共同項目新增 / 編輯 / 刪除。

---

## 私人確認清單

目前已完成：

- 入口位於共同檢查清單下方。
- Guest 不顯示入口。
- 本機 ownership 使用 `userEmail + tripId`。
- 可新增、編輯、刪除、勾選。
- 可顯示私人準備進度。
- 編輯時支援 Enter 儲存、Escape 取消。
- React StrictMode 下不會重複新增項目。
- 登出或未登入停在私人清單 / 記帳本時會導回行程頁。
- `trip_editor` / `super_admin` 可同步自己的私人清單到 Supabase。
- `super_admin` / `trip_editor` 不可讀取其他使用者私人清單。
- 雲端 item 以 `client_item_id` 對應本機 `private_...` item id。
- UI 顯示同步中、已同步、同步失敗。

私人確認清單資料流：

```text
PrivateChecklistPage
↓
usePrivateChecklistState
↓
privateChecklistService / privateChecklistCloudService
↓
localStorage + Supabase
```

# 四、下一步

下一步請接：

```text
共同檢查清單最小雲端同步
```

建議範圍：

1. 新增 shared checklist cloud service。
2. App 第一次進共同清單時，若 Supabase 沒有 `scope = shared` checklist，就用 Trip JSON `checklistData` 初始化。
3. 共同清單之後優先讀 Supabase shared checklist。
4. `guest` / `user` 可見不可勾選。
5. `trip_editor` / `super_admin` 可勾選並同步 `is_checked`。
6. 暫時不做共同項目新增、編輯、刪除。

暫緩：
