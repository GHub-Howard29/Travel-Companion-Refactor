# 二、已完成重點

## 共同檢查清單

目前使用：

```text
Trip JSON checklistData
↓
ChecklistPage
↓
useChecklistState
↓
sharedChecklistCloudService / checklistService
↓
checklistStorage
↓
Supabase + localStorage fallback
```

目前能力：

- App 第一次進共同清單時，若 Supabase 沒有 `scope = shared` checklist，會用 Trip JSON `checklistData` 初始化雲端共同清單。
- 共同清單優先讀 Supabase shared checklist。
- `trip_editor` / `super_admin` 可勾選並同步 `is_checked`。
- `guest` / `user` 可見不可勾選，且永遠看到乾淨未勾選版本，方便自行對照準備。
- 若雲端 shared checklist 已存在但缺少 Trip JSON seed item，`trip_editor` / `super_admin` 進頁時會補齊缺少 rows。
- 共同清單已支援 App 內新增 / 編輯 / 刪除。
- 共同清單管理操作已收合至「管理 / 退出」模式，避免日常瀏覽時誤觸。

尚未完成：

- Checklist Pending Queue。
- Conflict Resolution。

## 私人確認清單

目前已完成：

- 入口位於共同檢查清單下方。
- Guest 不顯示入口。
- 本機 ownership 使用 `userEmail + tripId`。
- 可新增、編輯、刪除、勾選。
- 可顯示私人準備進度。
- 編輯時支援 Enter 儲存、Escape 取消。
- React StrictMode 下不會重複新增項目。
- 登出或未登入停在私人清單 / 記帳本時會導回行程頁。
- `trip_editor` / `super_admin` 可同步自己的私人清單到 Supabase。
- `super_admin` / `trip_editor` 不可讀取其他使用者私人清單。
- 雲端 item 以 `client_item_id` 對應本機 `private_...` item id。
- UI 顯示同步中、已同步、同步失敗。
- 第一次進私人清單時，若雲端私人清單已建立但沒有內容，顯示「雲端私人清單已準備好，新增項目後會同步」，不顯示同步失敗。

私人確認清單資料流：

```text
PrivateChecklistPage
↓
usePrivateChecklistState
↓
privateChecklistService / privateChecklistCloudService
↓
localStorage + Supabase
```

# 七、開發規則

- 平常小步實作不逐次更新文件。

- 文件或程式修改完成且驗證通過後，Codex 直接建立 commit。

- Codex 使用專業術語時，必須附上對應本專案功能的解釋。例如 `RLS` 要說明為「Supabase 資料列權限，用來限制誰能讀寫特定 Trip 或使用者資料」。
