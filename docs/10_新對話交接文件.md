# 二、已完成重點

## 共同檢查清單

目前使用：

```text
Trip JSON checklistData
↓
ChecklistPage
↓
useChecklistState
↓
sharedChecklistCloudService / checklistService
↓
checklistStorage
↓
Supabase + localStorage fallback
```

目前能力：

- App 第一次進共同清單時，若 Supabase 沒有 `scope = shared` checklist，會用 Trip JSON `checklistData` 初始化雲端共同清單。
- 共同清單優先讀 Supabase shared checklist。
- `trip_editor` / `super_admin` 可勾選並同步 `is_checked`。
- `guest` / `user` 可見不可勾選，且永遠看到乾淨未勾選版本，方便自行對照準備。
- 若雲端 shared checklist 已存在但缺少 Trip JSON seed item，`trip_editor` / `super_admin` 進頁時會補齊缺少 rows。
- 共同清單已支援 App 內新增 / 編輯 / 刪除。
- 共同清單管理操作已收合至「管理 / 退出」模式，避免日常瀏覽時誤觸。

同步策略：

- 共同檢查清單已具備本機 `localStorage` 持久化與 Supabase 最小雲端同步。
- 目前不另做 Checklist Pending Queue，也就是不建立離線或同步失敗時的待上傳操作佇列。
- 目前不另做複雜 Conflict Resolution；若兩個使用者幾乎同時修改同一筆共同清單項目，以最後成功上傳到 Supabase 的那筆資料為準，下一次讀取時再同步回本機。

# 七、開發規則

- 平常小步實作不逐次更新文件。
- Product Owner 要求整理文件、整理交接文件、準備 commit 或要求 commit 時，再集中整理文件。
- 文件或程式修改完成且驗證通過後，Codex 直接建立 commit。
- commit message 使用繁體中文。
- commit 完成後，Codex 回報實際使用的中文 commit message。
- commit 前不可提交 `dist/`、`.env` 或 secrets，除非 Product Owner 明確要求。
- Codex 使用專業術語時，必須附上對應本專案功能的解釋。例如 `RLS` 要說明為「Supabase 資料列權限，用來限制誰能讀寫特定 Trip 或使用者資料」。
