# Bug 修正紀錄

## 2026/06/30

### BUG001

問題：

離線新增帳目後，Pending（待同步）數量沒有立即更新。

例如：

- P1 → 顯示 1（正常）
- P2（離線新增）→ 應顯示 2，但仍顯示 1。

恢復網路、重新整理或 F5 後，數量才恢復正常。

原因：

資料新增成功、Pending Queue 正常建立，但 Pending Badge 沒有立即重新計算，屬於 UI 更新時機問題。

修正方式：

調整 Pending Badge 更新流程，新增、修改、刪除帳目時立即重新計算 Pending 數量。

狀態：

✅ 已修正（V2.0）

---

### BUG002

問題：

同步完成後，如果修改附件，Pending 數量未立即更新。

原因：

附件修改未同步更新 Pending 狀態。

修正方式：

附件新增、修改、刪除時，同步更新 Pending Queue。

狀態：

✅ 已修正（V2.0）

## 2026/07/06

### BUG003

問題：

手機點選附件時，只出現拍照功能，沒有提供相簿或檔案選擇。

電腦則只有檔案選擇，沒有拍照功能。

2026/07/11 補充：

當時新增記帳本資料時，手機端附件入口需重新確認；預期行為為行動裝置同時支援「使用手機拍照」與「選擇既有照片 / 檔案」。

2026/07/12 補充：

已由 BUG011 的修正取代單一檔案入口做法，新增與編輯帳目都改為「拍照」與「相簿」兩個明確入口。

原因：

不同裝置對 `<input type="file">` 的支援方式不同，目前尚未針對行動裝置進行最佳化。

修正方式：

- 新增帳目與編輯帳目兩個附件入口拆成「拍照」與「相簿」。
- 「拍照」入口使用 `capture="environment"`。
- 「相簿」入口保留一般 `accept="image/*"`，維持既有照片壓縮與同步流程。

狀態：

✅ 已修正（2026/07/12）/ 需手機實機回歸

### BUG006

問題：

2026/07/11 補充：

目前仍只能更換附件，無法在編輯帳目時單獨刪除既有附加檔案。

原因：

目前編輯畫面未提供「移除附件」功能。

- 儲存後保留帳目資料，只清除附件欄位、本機附件與雲端附件連結。

狀態：

✅ 已修正（2026/07/11）/ 需附件刪除回歸

## 2026/07/11

### BUG008

問題：

領隊導遊聯絡資訊 / 自駕租車須知與其他資訊目前可直接看到新增 / 編輯入口，尚未完全比照其他 Travel Tool 的「先進入管理，再進下一層編輯」模式。

原因：

Checklist 與每日行程已改為瀏覽模式優先，但 Other Info / 特殊資訊頁仍保留舊的直接管理 UI。

修正方式：

- 領隊導遊聯絡資訊 / 自駕租車須知：預設只顯示卡片內容。
- 其他資訊：預設只顯示資料內容與分類。
- 有 `super_admin` 或符合編輯權限者，右上或適當位置顯示「管理」入口。
- 進入管理後才顯示新增、編輯、刪除。
- 管理層需提供「退出」按鈕。

狀態：

✅ 已修正（2026/07/11）/ 需瀏覽與管理模式回歸

## 2026/07/12

### BUG009

問題：

iOS 手機使用瀏覽器開啟時，不容易發現可安裝成 App 的入口；Android Chrome 會顯示安裝選項。

原因：

Android Chrome 支援 `beforeinstallprompt` 原生安裝提示；iOS Safari 不提供同等的程式化安裝 API，只能引導使用者透過分享選單加入主畫面。

修正方式：

- 行動瀏覽器若尚未以 App 模式開啟，顯示安裝建議提示。
- Android 可用原生安裝提示時，點選「立即安裝」觸發瀏覽器安裝流程。
- iOS 顯示「分享 → 加入主畫面」操作提示。
- 提示顯示 3 秒後自動縮回成小按鈕。

狀態：

✅ 已修正（2026/07/12）/ 需 iOS Safari 與 Android Chrome 實機回歸

### BUG010

問題：

記帳本新增帳目並附加照片後，iOS 手機容易出現照片同步失敗。

2026/07/12 補充：

同步失敗後，再次按同步可能顯示同步完成，但清單內仍顯示照片同步失敗，且無法點選開啟照片。

原因：

第一次同步失敗後，雲端狀態會變成 `upload_failed`。重新載入帳本時，程式只會替 `local_pending` 狀態恢復 IndexedDB 裡的本機照片 ID，沒有替 `upload_failed` 恢復，因此失敗項目看起來仍有附件名稱，但實際上失去可重試上傳的本機照片連結。

原始 iOS 同步失敗原因仍需實機追查，可能與 iOS Safari / PWA 在 IndexedDB 暫存照片、Canvas 壓縮、背景切換或 Supabase Storage 上傳期間的網路狀態有關。

修正方式：

- `upload_failed` 與 `local_pending` 一樣會在重新載入時嘗試恢復 IndexedDB 本機照片 ID。
- 再次同步時，`upload_failed` 若仍找得到本機照片，會重新上傳。
- 若找不到本機照片，不再誤顯示同步成功，改提示需重新拍照或選擇照片。
- 同步結果提示改為顯示成功與失敗筆數。

狀態：

✅ 已部分修正（2026/07/12）/ 原始 iOS 上傳失敗原因仍需實機蒐集錯誤訊息

### BUG011

問題：

記帳本照片附件在 Android Chrome 只出現相簿，沒有明確的拍照入口；iOS 會出現拍照或相簿選擇。

原因：

不同瀏覽器對單一 `<input type="file" accept="image/*">` 的行動裝置選擇器呈現不同，Android 不一定自動提供相機。

修正方式：

- 新增帳目與編輯帳目都拆成兩個附件入口：「拍照」與「相簿」。
- 「拍照」入口使用 `capture="environment"`，優先開啟後鏡頭。
- 「相簿」入口保留一般 `accept="image/*"`，供使用既有照片。

狀態：

✅ 已修正（2026/07/12）/ 需 iOS 與 Android 實機回歸

---

### BUG012

問題：

多人共用帳本時，一人新增或刪除單筆帳目，另一人的手機不會即時更新，需重新整理或重新進入才看得到。

原因：

記帳本目前只在進入頁面、指定操作後重新讀取資料，尚未監聽 Supabase 資料變更，也沒有固定時間刷新。

修正方式：

- 共用帳本加入 Supabase Realtime 監聽 `expenses` table 的同旅程變更。
- 若 Realtime 未啟用或網路狀態不穩，保留 30 秒定時刷新備援。
- 視窗重新取得焦點、App 回到前景、恢復連線時立即刷新帳本。

狀態：

✅ 已修正（2026/07/12）/ 若 Supabase Realtime 未開啟，會以 30 秒輪詢更新
