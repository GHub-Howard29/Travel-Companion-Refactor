# Bug 修正紀錄

本文件記錄專案開發過程中所發現的 Bug、原因分析、修正方式及目前狀態。

---

## 2026/06/30

### BUG001

問題：

離線新增帳目後，Pending（待同步）數量沒有立即更新。

例如：

- P1 → 顯示 1（正常）
- P2（離線新增）→ 應顯示 2，但仍顯示 1。

恢復網路、重新整理或 F5 後，數量才恢復正常。

原因：

資料新增成功、Pending Queue 正常建立，但 Pending Badge 沒有立即重新計算，屬於 UI 更新時機問題。

修正方式：

調整 Pending Badge 更新流程，新增、修改、刪除帳目時立即重新計算 Pending 數量。

狀態：

✅ 已修正（V2.0）

---

### BUG002

問題：

同步完成後，如果修改附件，Pending 數量未立即更新。

原因：

附件修改未同步更新 Pending 狀態。

修正方式：

附件新增、修改、刪除時，同步更新 Pending Queue。

狀態：

✅ 已修正（V2.0）

---

## 2026/07/06

### BUG003

問題：

手機點選附件時，只出現拍照功能，沒有提供相簿或檔案選擇。

電腦則只有檔案選擇，沒有拍照功能。

2026/07/11 補充：

新增記帳本資料時，手機端附件入口仍需重新確認。預期行為為行動裝置同時支援「使用手機拍照」與「選擇既有照片 / 檔案」，不可只出現單一路徑。

原因：

不同裝置對 `<input type="file">` 的支援方式不同，目前尚未針對行動裝置進行最佳化。

預計修正：

- V3.5 改善 Mobile UX。
- 需檢查新增帳目與編輯帳目兩個附件入口是否使用一致的 file input / capture 設定。
- 手機端需補齊拍照與相簿 / 檔案選擇流程。

狀態：

🚧 待修正 / 需實機回歸

---

### BUG004

問題：

Checklist 勾選狀態會影響其他旅程。

例如：

A 行程勾選完成後，B 行程也會顯示相同勾選結果。

原因：

Checklist 尚未依旅程建立獨立資料。

預計修正：

每個 Trip 建立自己的 Checklist 資料。

狀態：

🚧 待修正（V3.4）

---

### BUG005

問題：

Checklist 勾選後，重新整理（F5）會全部消失。

原因：

目前僅使用 React State 保存。

尚未建立 IndexedDB 或 localStorage 持久化。

預計修正：

新增 Checklist Persistence。

狀態：

🚧 待修正（V3.4）

---

### BUG006

問題：

已記錄帳目於編輯模式下，附件只能覆蓋，無法直接刪除。

2026/07/11 補充：

目前仍只能更換附件，無法在編輯帳目時單獨刪除既有附加檔案。此問題需保留為未完成缺口。

原因：

目前編輯畫面未提供「移除附件」功能。

預計修正：

新增附件刪除按鈕。

刪除附件後保留原帳目資料。

狀態：

🚧 待修正（V3.3）/ 需實作與回歸

## 2026/07/11

### BUG008

問題：

領隊導遊聯絡資訊 / 自駕租車須知與其他資訊目前可直接看到新增 / 編輯入口，尚未完全比照其他 Travel Tool 的「先進入管理，再進下一層編輯」模式。

原因：

Checklist 與每日行程已改為瀏覽模式優先，但 Other Info / 特殊資訊頁仍保留舊的直接管理 UI。

預計修正：

- 領隊導遊聯絡資訊 / 自駕租車須知：預設只顯示卡片內容。
- 其他資訊：預設只顯示資料內容與分類。
- 有 `super_admin` 或符合編輯權限者，右上或適當位置顯示「管理」入口。
- 進入管理後才顯示新增、編輯、刪除。
- 管理層需提供「退出」按鈕。

狀態：

🚧 待修正
