# 模組架構說明

> Version：V3.0
>
> 最後更新：2026-07

---

# 一、目的

本文件說明 Travel Companion V3 的模組架構，以及各模組之間的責任與關係。

Travel Companion 採用「Trip（旅程）」作為產品核心。

所有 Travel Tool（旅行工具）皆依附於 Trip，Foundation 則提供所有 Tool 共用的能力。

本文件僅說明模組架構，不討論功能規格與資料庫設計。

---

# 二、整體架構

```text
Travel Companion
│
├── Foundation（共同能力）
│
└── Trip（旅程）
    │
    ├── 旅程資訊
    ├── 記帳本
    ├── 共同清單
    ├── 私人確認清單
    └── 其他資訊
```

目前所有功能皆依附於 Trip。

新增功能時，應優先以新增 Travel Tool 的方式擴充，而非修改既有架構。

---

# 三、Foundation（共同能力）

// 提供所有 Travel Tool 共用能力 //

Foundation 不提供任何產品功能。

Foundation 統一提供：

- App Context
- Workspace
- Permission
- Role
- Storage
- Sync

所有 Travel Tool 均透過 Foundation 使用共同能力。

---

# 四、Trip（旅程）

// Travel Companion 的產品核心 //

Trip 為所有資料與功能的核心。

每一個 Trip 都是一個獨立的旅行空間。

Trip 負責管理：

- 旅程本身的基本資訊
- Trip 內所有 Travel Tool
- Trip 成員
- Trip 權限

所有 Travel Tool 均依附於 Trip。

---

# 五、Travel Tool（旅行工具）

// 所有功能皆依附於 Trip //

目前已規劃的 Travel Tool：

- 旅程資訊
- 記帳本
- 共同清單
- 私人確認清單
- 其他資訊

所有 Travel Tool 均遵循相同的設計原則。

各 Tool 的詳細功能與規格請參閱：

《06_功能規格.md》

---

# 六、模組合作方式

```text
Foundation
        │
        ▼
      Trip
        │
        ├── 旅程資訊
        ├── 記帳本
        ├── 共同清單
        ├── 私人確認清單
        └── 其他資訊
```

Foundation 提供共同能力。

Trip 管理旅程。

Travel Tool 專注於各自的產品功能。

各模組保持低耦合，避免直接依賴彼此。

---

# 七、未來擴充

當新增旅行工具時：

1. 先完成《06_功能規格.md》
2. 再建立新的 Travel Tool
3. 最後整合至 Trip

產品開發順序請參閱：

《02_產品開發路線圖.md》

---

# 八、設計原則

Travel Companion 採用以下模組設計原則：

- Trip 為產品核心。
- Foundation 提供共同能力。
- 所有 Travel Tool 皆依附於 Trip。
- Travel Tool 採用統一規格設計。
- 新增功能以擴充為優先，不修改既有架構。
- 保持低耦合、高可維護性。
