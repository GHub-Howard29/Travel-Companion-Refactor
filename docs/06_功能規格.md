# 功能規格

> Version：V3.0

---

# 一、目的

本文件定義 Travel Companion V3 各旅行工具（Travel Tool）的產品功能規格。

每個旅行工具均依附於 Trip（旅程）之下，並遵循統一的設計規格。

本文件描述的是產品功能與規格，不負責記錄開發進度；開發版本與完成狀態請參閱《02_產品開發路線圖》。

---

# 二、旅行工具定義

旅行工具（Travel Tool）係指依附於 Trip（旅程）之下，提供單一產品功能的功能模組。

目前已規劃的旅行工具：

- 旅程資訊
- 記帳本
- 共同清單
- 私人確認清單
- 其他資訊

未來將依產品需求持續新增新的旅行工具。

---

# 三、Trip 架構

```text
Trip（旅程）
│
├── 旅程資訊
│
├── 記帳本
│
├── 共同清單
│
├── 私人確認清單
│
└── 其他資訊
```

所有旅行工具皆依附於 Trip。

---

# 四、旅行工具共通規格

所有旅行工具均須定義以下內容：

| 項目 | 說明 |
|------|------|
| 功能說明 | 此旅行工具的主要用途。 |
| 儲存位置 | localStorage、IndexedDB、Supabase 等。 |
| 資料擁有者 | Trip 或個人。 |
| 適用對象 | 哪些使用者可使用此旅行工具。 |
| 權限規則 | Viewer、Editor、Admin 等權限規則。 |
| 是否可編輯 | 是否允許修改內容。 |
| 同步方式 | 不同步、本機同步、雲端同步等。 |
| 生命週期 | 建立、保存與刪除時機。 |

# 五、旅程資訊

## 功能說明

管理旅程本身的基本資訊。

## 旅行工具規格

| 項目 | 內容 |
|------|------|
| 儲存位置 | Supabase |
| 資料擁有者 | Trip |
| 適用對象 | Trip 成員 |
| 權限規則 | 依 Trip 權限管理 |
| 是否可編輯 | 目前僅建立旅程時建立，後續依功能規劃調整 |
| 同步方式 | 雲端同步 |

- 地點

### 成員管理

- 共用旅程
- 管理旅程成員

### 行程管理（未來）

- 每日行程
- 行程排序

# 六、記帳本

## 功能說明

管理旅程內所有收支資料及附件。

## 旅行工具規格

| 項目 | 內容 |
|------|------|
| 儲存位置 | IndexedDB／Supabase（依權限） |
| 資料擁有者 | Trip（一般 User 使用本機資料） |
| 適用對象 | Admin、Editor、User |
| 權限規則 | 依 Trip 權限管理 |
| 是否可編輯 | 可（依權限） |
| 同步方式 | Offline First，依權限同步 |
| 生命週期 | 建立 Trip 後即可使用，刪除 Trip 時依規則移除 |

## 管理資料

- Expense
- Attachment

## 功能能力

### 帳務管理

- 新增帳目
- 修改帳目
- 刪除帳目
- 修改記帳日期
- 日期排序
- 日期分頁
- 多幣別：目前支援 TWD、JPY、KRW、USD、EUR
- 付款人
- 若旅程設定 `participantEmailMap`，付款人可依登入 Email 鎖定為對應參與者；原本 `admin_users` 角色判斷仍維持獨立。
- 新增 / 編輯旅程時以 `名稱=Email` 維護參與者與登入 Email，名稱作為帳本付款人顯示，Email 用於鎖定登入者可使用的付款人。
- 分攤結算需先計算每位成員實際應分攤金額，再用已墊金額扣除應分攤金額，確保應收與應補總額一致。
- TWD / JPY / KRW 採整數分攤；若平均後有小數，應分攤者無條件進位負擔，代墊者承接零頭盈餘。
- USD / EUR 以分為最小單位結算。

### 附件管理

- 新增附件
- 修改附件
- 刪除附件
- 開啟附件
- 自動更新附件連結
- 新增 / 編輯時提供「拍照」與「相簿」兩個明確入口
- 附件同步失敗後可重試，重試時需保留本機附件暫存

目前已完成補強：

- 編輯既有帳目時，若該帳目已有附件，已提供「移除附件」功能，且刪除附件時保留帳目本身。
- 新增帳目與編輯帳目的附件入口已一致支援手機拍照與選擇既有照片 / 檔案。
- iOS 附件同步失敗、重試與附件連結開啟仍需實機回歸。

### 同步管理

- Offline First
- Pending Queue
- Pending Badge
- 手動同步
- 自動同步
- 帳目同步
- 附件同步
- Supabase realtime 更新
- 30 秒固定刷新 fallback
- 頁面重新聚焦、網路恢復、頁面可見時重新刷新

### 匯出

- Excel 匯出
- 匯出記帳日期
- 匯出附件網址

# 七、共同清單

## 功能說明

管理旅程所有成員共同使用的清單。

## 旅行工具規格

| 項目 | 內容 |
|------|------|
| 儲存位置 | Trip JSON seed + localStorage 進度 |
| 資料擁有者 | Trip |
| 適用對象 | Guest、User、Trip Editor、Super Admin |
| 權限規則 | Guest / User 可見不可勾選；Trip Editor / Super Admin 可勾選 |
| 是否可編輯 | V3-1 不提供 App 內新增 / 編輯 / 刪除項目 |
| 同步方式 | 目前本機保存勾選狀態；下一步接 Supabase shared checklist 最小同步 |
| 生命週期 | 建立 Trip 後即可使用，刪除 Trip 時一併刪除 |

## 管理資料

- Checklist
- Checklist Item

## 功能能力

### 清單管理

- 讀取 Trip JSON seed 清單

### 項目管理

- V3-1 尚不提供 App 內新增、修改、刪除項目

### 完成狀態

- 勾選完成
- 依 `tripId` 保存於 localStorage
- 進度計算忽略已不存在的 seed item id

### 下一步同步範圍

- 若雲端沒有 shared checklist，使用 Trip JSON seed 初始化。
- 共同清單優先讀 Supabase shared checklist。
- `trip_editor` / `super_admin` 可同步勾選狀態。
- 暫不提供 App 內新增、編輯、刪除共同項目。

### 顯示

- 排序
- 篩選
- 搜尋

# 八、私人確認清單

## 功能說明

管理個人準備事項。

私人確認清單是獨立 Travel Tool，入口位於左上角功能選單中共同檢查清單下方。

## 旅行工具規格

| 項目 | 內容 |
|------|------|
| 儲存位置 | localStorage + Supabase（依權限） |
| 資料擁有者 | 個人 + Trip |
| 適用對象 | User、Trip Editor、Super Admin |
| 權限規則 | Guest 不可使用；登入使用者僅可查看與管理自己的私人清單 |
| 是否可編輯 | 可新增、編輯、刪除、勾選自己的項目 |
| 同步方式 | User 僅本機；Trip Editor / Super Admin 可同步自己的私人清單 |
| 生命週期 | 依 `userEmail + tripId` 建立本機資料 scope |

## 管理資料

- Private Checklist
- Private Checklist Item

## 功能能力

### 項目管理

- 新增私人項目
- 編輯私人項目
- 刪除私人項目

### 完成狀態

- 勾選完成
- 顯示私人準備進度
- F5 後保留本機資料
- 顯示同步中、已同步、同步失敗狀態

### 權限限制

- Guest 不顯示私人確認清單入口。
- User 可使用自己的本機私人清單，但不做雲端同步。
- Trip Editor / Super Admin 可同步自己的私人清單，且不得讀取其他使用者私人清單。
- 本機 ownership 使用 `userEmail + tripId`。
- 雲端 ownership 使用 `owner_user_id = auth.uid()` 與 `trip_id`。
- 雲端 item 使用 `client_item_id` 對應本機 item id。

# 九、其他資訊

## 功能說明

管理旅程參考資訊與旅行資訊。

## 旅行工具規格

| 項目 | 內容 |
|------|------|
| 儲存位置 | 預設資料 + localStorage 本機覆寫 |
| 資料擁有者 | Trip |
| 適用對象 | Guest、User、Trip Editor、Super Admin |
| 權限規則 | V3-1 由 Super Admin 維護；其他角色唯讀 |
| 是否可編輯 | 目前 App 內可新增、編輯、刪除，資料保存於 localStorage；後續需改為先進入管理層才顯示編輯操作 |
| 同步方式 | V3-1 不同步 |
| 生命週期 | 預設資料隨 Build 發布，本機覆寫保存在使用者裝置 |

## UI 管理規格

- 預設畫面以瀏覽資訊為主。
- 有管理權限者顯示「管理」入口。
- 點選「管理」後才顯示新增、編輯、刪除。
- 管理層需提供「退出」按鈕回到瀏覽模式。
- 領隊導遊聯絡資訊與自駕租車須知需比照其他資訊採相同模式。
