Function List — 檔案：App.tsx

applyTripDefaults: 套用已選行程的預設（參與者、表單幣別、activeCurrency）。
getBasePath: 回傳網站根路徑（處理 /Travel-Companion 前綴）。
handleAttachmentSelection: 處理使用者選取附件並嘗試壓縮（設定 state）。
handleGoogleLogin: 使用 Supabase 進行 Google OAuth 登入。
handleLogout: 登出並清除相關 localStorage 與 state。
loadTripAndAuthData (async, useEffect 內): 載入選定行程詳細、管理 admin_profile 與載入/快取帳本資料。
syncOfflineData (async, useEffect 內): 自動將離線暫存的共用帳本資料同步至雲端。
handleAddExpense: 表單提交：新增支出（含附件處理、離線/個人帳本/雲端分流邏輯）。
createLocalExpenseItem (內部): 建立本地暫存的 Expense 物件並儲存本機附件。
saveToPersonalBook (內部): 將項目存到個人本機帳本。
saveToOfflineSandbox (內部): 將項目存到離線暫存箱（待同步）。
replaceExpenseInStorage: 在 localStorage 中替換/更新單筆帳目。
removeExpenseFromStorage: 從 localStorage 中移除單筆帳目。
cancelPendingDelete: 取消待確認刪除（清除計時器與 state）。
finalizeDeleteExpense (async): 完成刪除流程（雲端刪除、移除附件、恢復機制）。
startEditExpense: 啟動編輯某筆帳目（初始化編輯草稿與檔案）。
cancelEditExpense: 取消編輯並重置編輯草稿。
handleSaveEditExpense (async): 儲存編輯的帳目（處理本地/雲端更新與附件）。
handleDeleteExpense: 刪除按鈕邏輯（兩次確認、離線條件檢查、觸發 finalize）。
getSignedAttachmentUrl (async): 由 Supabase 取得附件簽名下載網址。
handleOpenAttachment (async): 開啟附件（雲端簽名網址或本機 Blob）。
hasLocalAttachmentId: Type guard 檢查是否有 local_attachment_id。
handleSyncAttachments (async): 同步尚未上傳的本機附件至 Supabase storage，並更新雲端紀錄。
buildExpenseXlsx (async): 使用 ExcelJS 建立要匯出的工作簿與超連結欄位。
getExportFileNameXlsx: 產生建議的 XLSX 檔名（含行程名稱與 scope）。
handleExportXlsx (async): 觸發匯出流程（支援 showSaveFilePicker 或 傳統下載）。
toggleChecklistItem: 切換檢查清單項目的完成狀態（更新 checkedItems）。
handleNavigate: 開啟 Google Maps 導航到指定地點。
handleScreenSelect: 切換側邊欄頁籤（包含未登入的防護與初始化 activeCurrency）。
