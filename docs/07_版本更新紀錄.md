# V3.x

## V3.0.0

開始：

2026/07/06

完成：

2026/07/11

狀態：

✅ 已完成，待合併主分支並部署

版本目標：

發布 Travel Companion V3 大改版。

本次為強制更新版本。原因：

- V1.0 已部署為舊正式版。
- V2.0 為重構版本，未部署。
- V3.0.0 整合 V2 重構成果與 V3-1 大量新功能。
- 不強制更新會讓使用者停留在舊版，無法體驗最新功能。

已完成內容：

- 旅行資訊（Reference）
- 其他資訊 / 參考資訊本機管理第一階段
- 共同檢查清單本機進度持久化
- 私人確認清單本機版
- 私人確認清單最小雲端同步
- 共同檢查清單最小雲端同步
- Trip 管理第一階段
- App 內新增 / 編輯旅程基本資料
- App 內新增 / 編輯 / 刪除每日行程資料第一版
- App 內新增 / 編輯 / 刪除共同清單項目第一版
- Trip Cloud schema / validation SQL 已執行並驗證
- PWA 版本更新提示
- 強制更新能力
- 權限矩陣第一階段落地

私人確認清單目前狀態：

- 入口位於共同檢查清單下方。
- Guest 不顯示入口。
- 登入使用者可新增、編輯、刪除、勾選自己的私人項目。
- 本機資料依 `userEmail + tripId` 保存於 localStorage。
- `trip_editor` / `super_admin` 可同步自己的私人項目到 Supabase。
- `super_admin` / `trip_editor` 不可讀取其他使用者私人清單。
- 尚未做 Pending Queue、Conflict Resolution。

共同檢查清單目前狀態：

- 使用 Trip JSON seed + Supabase shared checklist + localStorage fallback。
- App 第一次進共同清單時，若 Supabase 沒有 shared checklist，會以 Trip JSON `checklistData` 初始化。
- `trip_editor` / `super_admin` 可勾選並同步。
- `guest` / `user` 可見不可勾選，並看到乾淨未勾選版本。

狀態：

✅ 已完成，待合併主分支並部署

完成 V3.0.0 發布前回歸修正與發布流程補強。

主要更新內容：

- 補上其他資訊未來雲端同步所需的資料表與權限設計。

# Git Branch 建議

---

---

例如：

feature/permission

---
