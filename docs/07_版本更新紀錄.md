## V3.1.4

狀態：

✅ 程式與文件已完成，待 Product Owner 手動合併主分支並發布

完成：

2026/07/13

版本說明：

完成非強制更新提示恢復、旅程參與者欄位整併、常用幣別補強與帳本分攤規則修正。

本次為一般更新，非強制更新。

給使用者的白話更新內容：

- 發現新版本時會提示「馬上更新」與「稍後更新」；選擇稍後後，重新整理或重新開啟 App 仍會再次提醒。
- 選擇馬上更新時，App 會先清除瀏覽器暫存並重新載入最新版；已儲存旅程、清單、記帳與附件資料不會被清除。
- 馬上更新加入重新載入備援，降低 Android PWA 點擊後未自動更新的情況。
- iOS Safari 網頁模式也會顯示版本更新提示，方便未安裝 App 時更新。
- 更新提示補上未儲存資料警語，提醒使用者先儲存表單內容再更新。
- 新增 / 編輯旅程時，參與者名稱與登入 Email 合併為同一個 `名稱=Email` 欄位，減少重複輸入。
- 可編輯者 Email 補上權限說明，讓填寫者知道這些 Email 會取得旅程資訊與共用帳本編輯權限。
- 預設幣別調整為 `TWD NT$`、`JPY ¥`、`KRW ₩`、`USD $`、`EUR €`，並同步到記帳本可選幣別。
- TWD、JPY、KRW 分攤結算改為整數計算；不能整除時由應分攤者無條件進位負擔，代墊者承接零頭盈餘。

主要更新內容：

- `useAppUpdate` 的稍後更新不再寫入已看過版本，讓非強制更新在下一次 reload / 重新開啟時再次提示。
- 馬上更新會清除 Cache Storage 後再觸發 Service Worker 更新或 reload；不清除 `localStorage` 與 IndexedDB，避免移除已儲存旅行資料。
- 馬上更新在 Service Worker 更新流程外加入 1.5 秒 reload fallback，避免部分 Android PWA 未自動 reload。
- 更新提示顯示條件由僅安裝版 App 擴充為安裝版 App 或 iOS 裝置，讓 iOS Safari 網頁模式也能收到版本提示。
- `UpdatePrompt` 改為非強制更新顯示「馬上更新 / 稍後更新」，強制更新仍維持不可稍後。
- `TripEditorModal` 將 `participants` 與 `participantEmailMap` 在 UI 合併為 `名稱=Email`，送出時仍拆回原本資料結構，維持帳本付款人鎖定邏輯。
- 新增旅程時會以目前登入 Email 預填一筆參與者與可編輯者 Email。
- `SUPPORTED_CURRENCIES` 與新增旅程預設幣別同步為 TWD 優先，並加入 KRW / EUR。
- 記帳本分攤狀態新增每位成員的實際應分攤金額，摘要改為「平均參考」。
- TWD / JPY / KRW 採整數進位分攤；USD / EUR 仍以分為最小單位結算。

仍需回歸：

- 安裝版 PWA 從 V3.1.3 升到 V3.1.4 時，確認非強制更新提示、馬上更新清暫存與稍後更新再次提醒。
- 新增旅程時確認 `名稱=Email` 預填、儲存與付款人鎖定正確。
- 使用 TWD / JPY / KRW 建立不能整除的共同帳本，確認應收與應補總額一致。
- 確認 KRW / EUR 在新增帳目、編輯帳目、幣別頁籤與匯出資料中皆可正常使用。

- iOS 照片同步改為重新包裝照片資料，若第一次上傳失敗會改用另一種資料格式重試。
- 照片同步失敗時會顯示實際錯誤原因，方便判斷是授權、檔案格式或網路問題。

主要更新內容：

- iOS standalone PWA 登入改為先由使用者點擊開啟 Google 登入視窗，再以 Supabase OAuth URL 導向，避免兩步驗證期間被 PWA 容器中斷。
- iOS PWA 登入安全提示補上 YouTube / Google App 兩步驗證處理建議。
- Supabase Storage 照片上傳前會先以正確 MIME type 重新包裝 IndexedDB Blob。
- Blob 上傳失敗時會改用 ArrayBuffer 再重試一次，提高 iOS WebKit / PWA 照片同步相容性。
- 附件同步錯誤會保留第一筆失敗原因並顯示在提示中，方便後續實機排查。

仍需回歸：

- iOS PWA 使用 Google 登入，確認遇到 YouTube / Google App 兩步驗證時可依提示改用其他驗證方式或 Safari 網頁模式。
- iOS PWA 新增照片、同步照片與開啟照片連結。
- iOS 照片同步失敗提示是否能顯示可判讀的錯誤原因。
