# V3.x

## V3.0.0

開始：

2026/07/06

完成：

2026/07/11

狀態：

✅ 已完成，待合併主分支並部署

版本目標：

發布 Travel Companion V3 大改版。

本次為強制更新版本。原因：

- V1.0 已部署為舊正式版。
- V2.0 為重構版本，未部署。
- V3.0.0 整合 V2 重構成果與 V3-1 大量新功能。
- 不強制更新會讓使用者停留在舊版，無法體驗最新功能。

已完成內容：

- 旅行資訊（Reference）
- 其他資訊 / 參考資訊本機管理第一階段
- 共同檢查清單本機進度持久化
- 私人確認清單本機版
- 私人確認清單最小雲端同步
- 共同檢查清單最小雲端同步
- Trip 管理第一階段
- App 內新增 / 編輯旅程基本資料
- App 內新增 / 編輯 / 刪除每日行程資料第一版
- App 內新增 / 編輯 / 刪除共同清單項目第一版
- Trip Cloud schema / validation SQL 已執行並驗證
- PWA 版本更新提示
- 強制更新能力
- 權限矩陣第一階段落地

私人確認清單目前狀態：

- 入口位於共同檢查清單下方。
- Guest 不顯示入口。
- 登入使用者可新增、編輯、刪除、勾選自己的私人項目。
- 本機資料依 `userEmail + tripId` 保存於 localStorage。
- `trip_editor` / `super_admin` 可同步自己的私人項目到 Supabase。
- `super_admin` / `trip_editor` 不可讀取其他使用者私人清單。
- 尚未做 Pending Queue、Conflict Resolution。

共同檢查清單目前狀態：

- 使用 Trip JSON seed + Supabase shared checklist + localStorage fallback。
- App 第一次進共同清單時，若 Supabase 沒有 shared checklist，會以 Trip JSON `checklistData` 初始化。
- `trip_editor` / `super_admin` 可勾選並同步。
- `guest` / `user` 可見不可勾選，並看到乾淨未勾選版本。

## V3.1.0

狀態：

✅ 已完成，待合併主分支並部署

完成：

2026/07/11

版本說明：

完成 V3.0.0 發布前回歸修正與發布流程補強。

本次為一般更新，非強制更新。

主要更新內容：

- 修正版本更新提示，同版本或第一次開啟不再誤跳更新視窗。
- 其他資訊與自駕 / 領隊資訊改為先瀏覽、需要時再進入管理。
- 旅費記帳本可單獨移除已上傳的附件，不會刪除帳目。
- 手機新增或編輯照片附件時，可選擇拍照或既有照片。
- 補上其他資訊未來雲端同步所需的資料表與權限設計。

追加修正白話說明（2026/07/12）：

- 手機用瀏覽器開啟時，會提醒可以把 Travel Companion 安裝成 App；已安裝成 App 後才會看到版本更新提醒。
- 登入前會先顯示安全說明，讓使用者知道 App 只使用 Google Email 與登入狀態，不會讀取 Gmail、雲端硬碟、通訊錄、照片或行事曆。
- 記帳本新增或編輯附件時，拍照與相簿改成兩個清楚的按鈕，手機操作更直覺。
- 多人一起使用同一個帳本時，其他人新增或刪除帳目後，畫面會自動更新；即時同步不成功時，也會定時重新整理。
- 照片附件同步失敗後可以再試一次，系統會盡量保留手機裡暫存的照片，不會一失敗就把可重試資料清掉。
- 旅程可設定「參與者對應登入 Email」，記帳本付款人可依照登入帳號自動鎖定為對應的參與者。

追加修正技術內容（2026/07/12）：

- 版本資訊改為本次更新與版本歷史分開顯示，避免同一內容重複出現。
- 版本更新提示只在 App / PWA 安裝模式顯示，瀏覽器網頁開啟不再跳更新提示。
- 手機瀏覽器未安裝 App 時，新增安裝建議提示，提示 3 秒後自動收合。
- 登入前新增安全提示，說明登入後只使用 Google Email 與登入狀態。
- 領隊 / 開車特殊資訊頁補上管理入口。
- 記帳本附件入口改為明確分開「拍照」與「相簿」。
- 記帳本共用帳本加入 realtime 與 30 秒刷新 fallback。
- 附件同步失敗後重試會保留本機暫存，避免第一次失敗後第二次無資料可上傳。
- 新增參與者對應登入 Email，記帳本預設付款人可依登入帳號鎖定。

仍需實機回歸：

- iOS Safari 安裝建議與加入主畫面流程。
- Android Chrome 安裝提示流程。
- iOS / Android 記帳本拍照與相簿入口。
- iOS 附件同步失敗後重試與附件連結開啟。
- 兩台手機同一帳本新增 / 刪除後即時或 30 秒內更新。
- `super_admin` / `trip_editor` participantEmailMap 付款人鎖定。

## V3.1.1

狀態：

✅ 程式已完成，待執行 Other Info Supabase SQL 並實機回歸

完成：

2026/07/12

版本說明：

完成其他資訊 / 參考資訊最小雲端同步的前端基礎。

本次為一般更新，非強制更新。

給使用者的白話更新內容：

- 其他資訊開始接上雲端同步基礎，有權限者新增、編輯或刪除後，系統會嘗試同步到 Supabase。
- 開啟旅程時會優先讀取雲端其他資訊；如果雲端資料表尚未啟用或讀取失敗，仍會保留原本旅程資料，不影響使用。
- 刪除旅程時，會一併清理該旅程的其他資訊雲端資料。
- 這版仍保留本機與 Trip 內容備援，降低雲端同步初期的資料風險。

主要更新內容：

- 新增 `otherInfoCloudService`，負責讀取、寫入與軟刪除 `other_info_items`。
- 讀取 Trip detail 時會嘗試合併雲端 Other Info；失敗時退回既有 Trip content。
- Other Info 儲存時會比較刪除項目，新增 / 編輯走 upsert，刪除走 `deleted_at` 軟刪除。
- 刪除 Trip 時追加清理 `other_info_items`。
- App 版本設定與 PWA 版本 JSON 更新為 V3.1.1。

仍需回歸：

- 在 Supabase SQL Editor 執行 `docs/sql/005_other_info_cloud_schema.sql`。
- 執行 `docs/sql/006_other_info_cloud_validation.sql` 驗證資料表、權限與 grant。
- 使用 `super_admin` / `trip_editor` 實測其他資訊新增、編輯、刪除是否可同步。
- 使用 Guest / User 實測只能看到符合角色權限的其他資訊。
