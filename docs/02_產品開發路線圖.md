> Version：V3.0

# 一、產品願景

Travel Companion 的目標不是提供單一功能，而是打造一套完整的旅行管理平台。

所有功能皆以 Trip 為核心，逐步擴充各種 Travel Tool，陪伴使用者完成每一次旅行。

---

# 二、V3 核心目標

建立一套：

- 易閱讀
- 易維護
- 易擴充
- 易測試
- 支援長期演進

的產品架構。

---

# 三、產品核心架構

```text
Travel Companion
        │
        ▼
      Trip
        │
        ▼
 Travel Tools
```

Trip 為產品核心。

所有功能皆依附於 Trip。

---

# 四、V3 開發階段

## 第一階段：Foundation（共同能力）

目標：

建立所有 Travel Tool 共用能力。

目前規劃：

- Workspace
- Current Trip
- Current User
- Role
- Permission
- Sync State
- Storage Layer

目前已新增 Trip 管理第一版：

- App 內新增 / 編輯旅程基本資料。
- 旅程清單改為 JSON seed + Supabase trips + localStorage fallback 合併。
- `super_admin` 可新增旅程。
- `trip_editor` 可編輯被指派旅程。
- 可編輯者 Email 由 Supabase `admin_users` 管理。
- Trip Cloud schema / validation 已透過 Supabase connector 執行並驗證。

完成後，所有 Travel Tool 共用同一套基礎能力。

---

## 第二階段：Travel Tool 模組化

逐步完成各功能模組。

目前規劃：

- Expense
- Checklist（已完成共同 / 私人清單的最小雲端同步）
- Reference（V3-1 第一階段已完成 localStorage 簡易管理）

完成獨立模組化。

各 Tool 不互相依賴。

---

## 第三階段：新功能擴充

依產品需求新增：

- Reservation
- Packing
- Travel Journal
- Travel Information
- 更多旅行工具

新增功能時，不修改既有 Tool。

---

## 第四階段：智慧功能

逐步加入 AI 功能。

例如：

- AI 行程建議
- AI 花費分析
- AI 打包建議
- AI 旅行助手

AI 屬於新的 Travel Tool。

不影響既有架構。

# 六、Checklist 規劃

目前規劃：

- Public Checklist
- Private Checklist

兩者皆依附於 Trip。

目前已完成：

- 共同檢查清單最小雲端同步。
- 私人確認清單最小雲端同步。
- `guest` / `user` 查看共同清單時顯示乾淨未勾選版本，作為自己的準備對照。
- `trip_editor` / `super_admin` 可同步共同清單勾選狀態。

後續保留：

- 共同項目 App 內新增 / 編輯 / 刪除。
- Pending Queue（離線或同步失敗時暫存待上傳操作的佇列）。
- Conflict Resolution（本機與雲端同一筆旅行資料都被修改時的合併或取捨規則）。

下一步建議：

Checklist 先視為 V3-1 最小同步完成；Reference / Other Info 權限過濾已完成，下一步轉往 Supabase schema / RLS 與雲端同步。

近期 UI 補強：

- 共同檢查清單與私人確認清單已改為瀏覽優先，透過「管理」進入新增、編輯、刪除層。
- 領隊導遊聯絡資訊 / 自駕租車須知與其他資訊已採同一個「管理 / 退出」模式，避免管理員日常瀏覽時誤觸。
- 旅費記帳本已補上既有附件單獨移除功能，附件移除不會刪除帳目資料。
- 旅費記帳本新增與編輯附件入口已改為「拍照」與「相簿」兩個明確入口，避免不同手機瀏覽器只顯示單一路徑。

# 七、Reference 規劃

Reference 提供：

旅行資訊整理。

例如：

- 地址
- 網站
- 注意事項
- 行前資訊

預設資料由 seed data 提供。

V3-1 允許使用者在 APP 內做簡易本機管理。

V3-1 第一階段已完成：

- Trip sidebarConfig 接入「其他資訊」
- OtherInfoPage 簡易管理 UI
- Folder chips 與資料筆數
- 自由行 / 跟團第一批 seed data
- seed data + localStorage 合併讀取
- 新增、編輯、刪除 item
- 編輯 seed item 時以 localStorage 覆蓋
- 刪除 seed item 時以 `isDeleted` 標記覆蓋
- `http` / `https` URL 自動轉為可點擊超連結

目前尚未完成：

- 雲端同步
- 多人協作
- 在 Supabase SQL Editor 執行並驗證 Other Info schema / RLS

下一步建議：

先執行 `docs/sql/005_other_info_cloud_schema.sql`，再執行 `docs/sql/006_other_info_cloud_validation.sql` 驗證，最後實作最小雲端同步。

未來可持續擴充。

# 八、權限規劃

目前規劃：

- 未登入使用者
- 一般登入使用者
- Trip Editor
- Super Admin

所有權限皆由 Foundation 管理。

目前 Trip 管理權限：

- Guest：可瀏覽旅程。
- 一般登入使用者：可使用個人功能，但不可新增 / 編輯共享旅程。
- Trip Editor：可編輯被指派旅程內容，不可管理可編輯者 Email。
- Super Admin：可新增旅程、編輯所有旅程、管理可編輯者 Email。

---

# 九、同步規劃

同步系統負責：

- Upload
- Download
- Pending Queue（離線或同步失敗時暫存待上傳操作的佇列）
- 自動同步
- 離線同步

Travel Tool 不直接處理同步。

# 十三、開發原則

每新增一個功能：

應優先建立新的 Travel Tool。

避免修改既有 Tool。

若需大量修改既有程式，應重新評估架構是否合理。

Travel Companion 將持續演進為：

一套完整的旅行管理平台。

以 Trip 為核心，

透過持續增加 Travel Tool，

逐步完成旅行所需的所有管理功能。
