# 產品開發路線圖

> Version：V3.0
>
> 最後更新：2026-07

---

# 一、產品願景

Travel Companion 的目標不是提供單一功能，而是打造一套完整的旅行管理平台。

所有功能皆以 Trip 為核心，逐步擴充各種 Travel Tool，陪伴使用者完成每一次旅行。

---

# 二、V3 核心目標

建立一套：

- 易閱讀
- 易維護
- 易擴充
- 易測試
- 支援長期演進

的產品架構。

---

# 三、產品核心架構

```text
Travel Companion
        │
        ▼
      Trip
        │
        ▼
 Travel Tools
```

Trip 為產品核心。

所有功能皆依附於 Trip。

---

# 四、V3 開發階段

## 第一階段：Foundation（共同能力）

目標：

建立所有 Travel Tool 共用能力。

目前規劃：

- Workspace
- Current Trip
- Current User
- Role
- Permission
- Sync State
- Storage Layer

完成後，所有 Travel Tool 共用同一套基礎能力。

---

## 第二階段：Travel Tool 模組化

逐步完成各功能模組。

目前規劃：

- Expense
- Checklist
- Reference（V3-1 第一階段已完成 localStorage 簡易管理）

完成獨立模組化。

各 Tool 不互相依賴。

---

## 第三階段：新功能擴充

依產品需求新增：

- Reservation
- Packing
- Travel Journal
- Travel Information
- 更多旅行工具

新增功能時，不修改既有 Tool。

---

## 第四階段：智慧功能

逐步加入 AI 功能。

例如：

- AI 行程建議
- AI 花費分析
- AI 打包建議
- AI 旅行助手

AI 屬於新的 Travel Tool。

不影響既有架構。

---

# 五、帳本規劃

目前分為：

## Local Ledger（本機帳本）

提供：

- 離線使用
- 個人資料管理

---

## Cloud Ledger（雲端帳本）

提供：

- 多人共同使用
- 雲端同步
- 權限管理

---

# 六、Checklist 規劃

目前規劃：

- Public Checklist
- Private Checklist

兩者皆依附於 Trip。

未來保留本機與雲端擴充能力。

下一步建議：

將目前「共同檢查清單」逐步重構為 Trip-scoped checklist tool，並為共用 / 私人清單與 local persistence 預留架構。

---

# 七、Reference 規劃

Reference 提供：

旅行資訊整理。

例如：

- 地址
- 網站
- 注意事項
- 行前資訊

預設資料由 seed data 提供。

V3-1 允許使用者在 APP 內做簡易本機管理。

V3-1 第一階段已完成：

- Trip sidebarConfig 接入「其他資訊」
- OtherInfoPage 簡易管理 UI
- Folder chips 與資料筆數
- 自由行 / 跟團第一批 seed data
- seed data + localStorage 合併讀取
- 新增、編輯、刪除 item
- 編輯 seed item 時以 localStorage 覆蓋
- 刪除 seed item 時以 `isDeleted` 標記覆蓋
- `http` / `https` URL 自動轉為可點擊超連結

目前尚未完成：

- 雲端同步
- 多人協作
- 權限過濾

未來可持續擴充。

---

# 八、權限規劃

目前規劃：

- 未登入使用者
- 一般登入使用者
- Trip Editor
- Super Admin

所有權限皆由 Foundation 管理。

---

# 九、同步規劃

同步系統負責：

- Upload
- Download
- Pending Queue
- 自動同步
- 離線同步

Travel Tool 不直接處理同步。

---

# 十、長期產品方向

未來所有新增功能皆遵循：

Trip
↓

Travel Tool

不得破壞既有產品架構。

---

# 十一、開發原則

每新增一個功能：

應優先建立新的 Travel Tool。

避免修改既有 Tool。

若需大量修改既有程式，應重新評估架構是否合理。

---

# 十二、最終目標

Travel Companion 將持續演進為：

一套完整的旅行管理平台。

以 Trip 為核心，

透過持續增加 Travel Tool，

逐步完成旅行所需的所有管理功能。
