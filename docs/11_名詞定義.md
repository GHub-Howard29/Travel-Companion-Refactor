# 名詞定義

> Version：V3.0
>
> 最後更新：2026-07

---

# 一、目的

本文件統一定義 Travel Companion V3 所使用的重要名詞。

所有設計、討論與文件皆以本文件為準。

---

# 二、產品核心

## Travel Companion

整個產品名稱。

提供旅行規劃、管理與紀錄等各種旅行工具。

---

## Trip（旅程）

代表一次完整的旅程。

Trip 為 Travel Companion 的產品核心。

所有功能皆依附於 Trip。

每個 Trip 擁有自己的旅行工具與相關資料。

---

## Workspace

目前正在操作中的 Trip。

所有畫面、資料、操作皆以目前 Workspace 為基準。

同一時間僅會有一個 Workspace。

---

# 三、Foundation

## Foundation

提供所有 Travel Tool 共用能力。

例如：

- Current Trip
- Current User
- Workspace
- Permission
- Role
- Sync State
- Storage

Foundation 不包含任何商業邏輯。

---

## Current Trip

目前正在操作的 Trip。

所有 Travel Tool 都透過 Current Trip 存取資料。

---

## Current User

目前登入的使用者。

包含登入資訊與基本資料。

---

## Sync State

目前同步狀態。

例如：

- 已同步
- 待同步
- 同步中
- 同步失敗

---

# 四、Travel Tool

## Travel Tool

依附於 Trip 的功能模組。

例如：

- Expense
- Checklist
- Reference
- Reservation（目前暫不規劃）
- Packing（目前暫不規劃）
- Travel Journal（目前暫不規劃）

每個 Tool：

- 獨立開發
- 獨立維護
- 獨立測試

共同使用 Foundation。

---

## Expense

旅行記帳工具。

管理：

- 收入
- 支出
- 類別
- 附件
- 匯出

---

## 共同清單

屬於旅程。

符合權限的旅程成員可共同使用。

V3-1 第一階段由 Trip JSON 提供 seed，勾選進度依 `tripId` 儲存在 localStorage。

未來可同步至雲端供旅程成員共同使用。

---

## 私人清單

屬於個人。

僅建立者可查看與管理。

V3-1 第一階段資料儲存在 localStorage。

不與其他旅程成員共享。

本機資料 ownership 使用：

```text
userEmail + tripId
```

未來雲端同步 ownership 使用：

```text
owner_user_id = auth.uid()
trip_id = 目前旅程 id
```

---

## Reference

旅行資訊整理工具。

例如：

- 地址
- 網站
- 注意事項
- 備忘錄

---

# 五、權限

## Role

角色。

代表使用者的身分。

例如：

- Guest
- User
- Trip Editor
- Super Admin

---

## Permission

權限。

決定目前角色可以執行哪些操作。

所有權限皆由 Permission Module 統一管理。

---

# 六、帳本

## Local Ledger

本機帳本。

資料僅保存在本機。

不需登入即可使用。

---

## Cloud Ledger

雲端帳本。

資料儲存在 Supabase。

提供多人協作與同步功能。

---

# 七、Storage

## Storage Layer

統一管理資料存取。

目前包含：

- localStorage
- IndexedDB
- Supabase

Travel Tool 不直接操作資料來源。

---

# 八、同步

## Pending

等待同步的資料。

尚未完成同步至雲端。

---

## Offline Mode

離線模式。

系統無法與雲端連線。

可依功能規則進行本機操作。

---

## Online Mode

連線模式。

可與雲端同步資料。

---

# 九、設計原則

## Product First

先思考產品，再思考技術。

---

## Trip First

Trip 為產品核心。

所有功能皆圍繞 Trip 建立。

---

## Foundation First

共同能力集中管理。

商業邏輯由各 Tool 自行管理。

---

## Simple First

避免過度設計。

優先考慮：

- 易閱讀
- 易維護
- 易擴充

---

## Travel Tool 設計規格

每個 Travel Tool 在設計時，都必須先定義：

- 儲存位置
- 資料擁有者
- 權限規則
- 是否可編輯
- 同步方式
- 生命週期

此六項規格為 Travel Companion V3 所有功能的共同設計標準。

---

# 十、文件維護原則

新增重要名詞時，

應同步更新本文件。

避免不同文件使用不同名稱。

本文件為 Travel Companion V3 唯一正式名詞定義來源。
